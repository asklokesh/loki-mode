Methodology

How Loki Mode approaches each benchmark problem using its multi-agent architecture.

Step 1

Architect Agent

Analyzes the problem description, identifies edge cases, and designs the solution approach.

Step 2

Engineer Agent

Implements the solution based on the architect's design, following coding best practices.

Step 3

QA Agent

Validates the implementation against test cases and checks for correctness.

Step 4

Review Agent

Checks code quality and verifies the solution handles all edge cases properly.

RARV Retry Cycle

Each problem follows the Reason - Act - Reflect - Verify cycle. If a solution fails validation, the system retries with updated context from the failure (up to 3 attempts for HumanEval). This self-correction mechanism is core to the multi-agent approach.

For SWE-bench, each problem is given up to 600 seconds. The agent reads the issue description, analyzes the relevant codebase, and generates a patch in unified diff format targeting the specific files that need modification.

Reproducing These Results

# Clone the repository
git clone https://github.com/asklokesh/loki-mode.git
cd loki-mode

# Setup benchmark datasets
./benchmarks/run-benchmarks.sh all

# Run HumanEval
./benchmarks/run-benchmarks.sh humaneval --execute --model opus

# Run SWE-bench Lite
./benchmarks/run-benchmarks.sh swebench --execute --model opus

# Run a subset first to test
./benchmarks/run-benchmarks.sh humaneval --execute --limit 10
